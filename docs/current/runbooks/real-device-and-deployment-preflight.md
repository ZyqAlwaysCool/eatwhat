# 真机调试与部署前检查

## 目标

为下一阶段的真机调试和部署提供统一前置检查清单。

## 真机调试前提

在进入真机调试前，当前应确认以下事项：

### 小程序端

- 已能在微信开发者工具中正常打开 `app/miniapp`
- 已能成功执行 `pnpm build:weapp`
- 已确认接入模式：
  - 域名模式：`MEAL_DECISION_API_BASE_URL` 指向真实后端地址
  - 云托管模式：`MEAL_DECISION_USE_CALL_CONTAINER=true` 且配置 `MEAL_DECISION_CLOUDRUN_SERVER`、`MEAL_DECISION_CLOUDRUN_ENV`
- 小程序已与目标云开发环境完成关联

### 后端

- 已能正常启动 FastAPI
- `/health` 可访问
- 已配置 `DATABASE_URL`（生产必需，必须指向 MySQL）
- 已准备好微信正式环境变量：
  - `WECHAT_APP_ID`
  - `WECHAT_APP_SECRET`
  - `WECHAT_LOGIN_MODE=official`

## 部署前提

当前推荐优先走微信云托管容器部署，部署前必须具备：

1. 已创建微信云托管环境
2. 已创建并部署后端服务（容器）
3. 已在云托管准备可用 MySQL，并生成 `DATABASE_URL`
4. 已在云托管配置后端环境变量（含 `DATABASE_URL`）
5. 小程序端已按云托管模式配置 `callContainer` 相关变量

如果选择公网域名模式，仍需额外具备域名、HTTPS 与微信合法域名配置。

## 当前不建议的做法

以下做法不建议作为正式真机方案：

- 直接使用裸 IP
- 在前端写死 `AppSecret`
- 前端直接调用微信 `code2Session`
- 让前端传入用户 ID 参与业务鉴权
- 生产环境继续依赖本地 SQLite 文件存储

## 当前推荐的上线前顺序

1. 先在本地完成 `./scripts/check`
2. 部署后端容器到微信云托管
3. 配置生产 `DATABASE_URL` 与微信登录变量
4. 小程序切换 `callContainer` 接入并真机验证
5. 切换 `WECHAT_LOGIN_MODE=official`
6. 用真机验证登录、候选池、历史、决策推荐

## 当前建议保留的验证项目

进入真机阶段后，至少验证以下链路：

- 首次进入小程序是否自动登录成功
- 新增店铺是否只写入当前用户的数据
- 历史页是否只显示当前用户自己的记录
- “直接帮我决定”是否能正常得到结果
- 重新打开小程序后会话是否仍然有效
- 会话失效后是否能自动重登

## 与下一阶段直接相关的事实

- 当前代码已经完成用户隔离，不需要再重新设计账号模型
- 当前真正的下一阶段重点是环境联通，而不是继续改核心技术栈
- 下一会话应优先关注：
  - 云托管环境关联
  - MySQL 连接配置
  - 镜像发布与回滚
  - 微信后台与真机验证
  - 真机联调问题排查
