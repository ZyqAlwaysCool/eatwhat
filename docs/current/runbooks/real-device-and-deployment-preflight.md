# 真机调试与部署前检查

## 目标

为下一阶段的真机调试和部署提供统一前置检查清单。

## 真机调试前提

在进入真机调试前，当前应确认以下事项：

### 小程序端

- 已能在微信开发者工具中正常打开 `app/miniapp`
- 已能成功执行 `pnpm build:weapp`
- 前端环境变量 `MEAL_DECISION_API_BASE_URL` 已指向真实可访问后端地址

### 后端

- 已能正常启动 FastAPI
- `/health` 可访问
- 已准备好微信正式环境变量：
  - `WECHAT_APP_ID`
  - `WECHAT_APP_SECRET`
  - `WECHAT_LOGIN_MODE=official`

## 部署前提

当前后端适合部署到阿里云 ECS，但部署前必须具备：

1. 一台可访问公网的 ECS
2. 一个域名
3. 该域名对应的 HTTPS 证书
4. 微信小程序后台配置好的 `request` 合法域名

## 当前不建议的做法

以下做法不建议作为正式真机方案：

- 直接使用裸 IP
- 在前端写死 `AppSecret`
- 前端直接调用微信 `code2Session`
- 让前端传入用户 ID 参与业务鉴权

## 当前推荐的上线前顺序

1. 先在本地完成 `./scripts/check`
2. 再部署后端到可访问环境
3. 配置域名与 HTTPS
4. 在微信后台登记合法域名
5. 切换 `WECHAT_LOGIN_MODE=official`
6. 修改小程序请求地址到正式域名
7. 用真机验证登录、候选池、历史、决策推荐

## 当前建议保留的验证项目

进入真机阶段后，至少验证以下链路：

- 首次进入小程序是否自动登录成功
- 新增店铺是否只写入当前用户的数据
- 历史页是否只显示当前用户自己的记录
- “直接帮我决定”是否能正常得到结果
- 重新打开小程序后会话是否仍然有效
- 会话失效后是否能自动重登

## 与下一阶段直接相关的事实

- 当前代码已经完成用户隔离，不需要再重新设计账号模型
- 当前真正的下一阶段重点是环境联通，而不是继续改核心技术栈
- 下一会话应优先关注：
  - 正式域名
  - HTTPS
  - ECS 部署方式
  - 微信后台配置
  - 真机联调问题排查
