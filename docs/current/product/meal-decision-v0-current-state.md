# 今天吃什么 V0 当前状态

## 当前产品定位

当前产品是一个微信小程序形态的“今天吃什么”决策工具，优先服务单人场景。

当前已经确认的产品边界：

- 目标不是平台级商家发现
- 目标是帮助用户在 1 到 3 分钟内完成一次吃饭决策
- 核心资产是用户自己的店铺池、方向池和反馈历史
- 决策链路优先级是“快速决定”高于“深度管理”

## 当前已实现的 V0 能力

### 1. 首页

- 可以直接发起一次决策
- 可以带少量条件进入决策
- 可以从最近一次决策快速再来一轮
- 可以跳转候选池与历史页

### 2. 条件页

- 支持品类、口味、预算、场景、用餐方式等条件
- 支持“尽量少打扰”的轻量筛选

### 3. 结果页

- 可以展示本次推荐结果
- 支持“就吃这个”
- 支持“换一个”
- 支持把当前方向补成店铺
- 结果页已经做过一轮减法，避免过多系统解释

### 4. 候选池页

- 支持新增店铺候选
- 支持查看店铺池
- 支持查看仍未被店铺覆盖的方向池
- 店铺名允许是简称、昵称，不要求标准化店名

### 5. 历史页

- 支持查看历史决策
- 支持记录接受、跳过、不喜欢、最近吃过、太贵了
- 支持把历史结果补回候选池

## 当前决策规则事实

当前不是纯随机抽取，而是以下链路：

1. 先按硬条件过滤
2. 再按口味、预算、场景、方式与历史反馈做轻量加权
3. 在可选项中执行加权抽选

当前还有一个关键产品规则：

- 如果某个品类下已经有 1 家及以上店铺，该品类本身不再作为独立方向参与抽取
- 没有被任何店铺覆盖的品类，才继续以“方向”身份参与抽取

因此当前的实际抽取池是：

- 店铺池
- 未被店铺覆盖的方向池

## 当前用户体系事实

当前已经接入微信小程序登录链路，并具备用户隔离：

- 用户进入小程序后自动登录
- 每个用户有自己的店铺池
- 每个用户有自己的自定义品类池
- 每个用户有自己的历史记录与反馈

## 当前未做的事情

截至当前阶段，以下内容仍未纳入 V0 已交付范围：

- 多人共同决策
- 外部商家平台接入
- 地图与距离排序
- 登录后用户资料页
- 后台管理系统
- AI 主导推荐解释
- 正式线上部署后的 PostgreSQL 迁移

## 当前可作为下阶段输入的事实

- V0 主流程已经可跑通
- 小程序当前重点已从“骨架搭建”转为“真机调试 + 部署联通”
- 下一阶段的重心应放在：
  - 真机网络联通
  - 正式微信登录联调
  - 域名、HTTPS、合法域名配置
  - 线上环境数据与部署策略
