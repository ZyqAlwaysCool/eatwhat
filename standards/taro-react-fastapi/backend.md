# 后端标准

## 技术栈

后端默认技术栈：

- Python
- FastAPI
- uv
- Ruff
- pytest

推荐持久化起步方案：

- 本地开发或早期验证：SQLite
- 进入稳定迭代后：PostgreSQL

## 工程组织

后端依赖管理默认使用仓库根目录下的 `pyproject.toml`。

后端代码默认放在：

- `app/api/main.py`
- `app/api/routes/`
- `app/api/schemas/`
- `app/api/services/`
- `app/api/domain/`
- `app/api/repositories/`

## 后端职责

第一版后端主要负责：

- 微信登录态换取与用户绑定
- 候选池数据读写
- 决策历史读写
- 规则配置承载
- 未来 AI 代理能力

## 服务边界

- 后端应提供稳定接口，不让小程序直接依赖第三方密钥
- 所有 AI 调用都应通过后端代理
- 用户体系以内部 `user` 为核心，不直接把微信展示信息当作业务主键

## 用户体系原则

默认登录路径：

1. 小程序调用 `wx.login`
2. 前端将 `code` 发给后端
3. 后端调用微信登录换取能力
4. 后端建立或绑定内部用户
5. 后端返回自己的登录态

第一版原则：

- 不强制手机号
- 不做复杂注册流程
- 先保证轻登录与可同步

## API 原则

- 路由保持轻量
- 请求和响应都使用明确 schema
- 错误返回保持一致
- 接口命名清晰，不把前端页面语义直接暴露为后端领域语义
