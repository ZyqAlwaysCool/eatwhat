# 工具与初始化

## 默认工具选择

前端默认工具：

- Node.js 18+
- pnpm
- Taro CLI

后端默认工具：

- Python 3.10+
- uv
- Ruff
- pytest

## 初始化原则

微信小程序项目不建议先用微信开发者工具的原生模板创建，再反向适配到 Taro。

当前默认初始化路径应为：

1. 使用 Taro CLI 创建小程序前端项目
2. 将前端项目放在 `app/miniapp`
3. 使用微信开发者工具“导入项目”打开 Taro 项目根目录进行预览

这样做的原因是：

- 代码结构由 Taro 主导，更适合 React 开发体验
- Codex 后续维护的是统一源码，而不是混合原生模板产物
- 与根脚本、验证和后续多端扩展更一致

## 官方依据

Taro 官方文档说明：

- 可以使用 `taro init myApp` 初始化项目
- 微信小程序可通过 `pnpm dev:weapp` 或 `pnpm build:weapp` 编译
- 微信开发者工具选择项目根目录进行预览

## 当前建议

因此，本项目的初始化建议明确为：

- 前端使用 Taro CLI 初始化
- 不使用微信开发者工具原生模板作为主初始化路径
- 微信开发者工具用于导入、预览、模拟器和真机测试
