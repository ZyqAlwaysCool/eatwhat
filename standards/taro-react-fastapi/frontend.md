# 前端标准

## 技术栈

前端默认技术栈：

- Taro
- React
- TypeScript
- Tailwind CSS
- pnpm

## 组件策略

不直接依赖 `shadcn/ui` 组件实现。

原因：

- `shadcn/ui` 主要面向 Web 运行环境
- 微信小程序不是普通 DOM 环境
- 直接移植组件行为与样式成本高，维护性差

推荐做法：

- 借鉴 `shadcn/ui` 的设计思路
- 自建一层轻量业务组件

第一批推荐组件：

- `Button`
- `Tag`
- `Card`
- `Section`
- `EmptyState`
- `ResultCard`
- `BottomActionBar`

## 页面开发原则

- 一个页面只解决一个主要任务
- 页面组件应保持扁平可读
- 条件收集、结果展示、候选池管理、历史回看应分页面处理
- 空状态、异常状态、无结果状态必须明确设计

## 状态管理原则

- 页面局部状态优先使用 React state
- 跨页面共享状态再引入统一 store
- 不要过早引入复杂状态管理框架
- 前端状态只保存展示与交互必要信息，规则计算尽量集中

## 样式原则

- 使用 Tailwind CSS 做基础样式组织
- 设计令牌集中管理，例如颜色、圆角、间距、阴影
- 避免样式散落为难以复用的魔法值
- 保持界面简洁、干净、聚焦

## 动效原则

- 动效只用于增强反馈，不用于制造噱头
- 优先使用简单过渡、淡入淡出、位移、缩放
- 避免大量复杂动画叠加
- 不依赖 AI 生图作为首版视觉基础

## 脚本约定

前端项目初始化后，默认应提供以下脚本：

- `pnpm dev:weapp`
- `pnpm build:weapp`
- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
