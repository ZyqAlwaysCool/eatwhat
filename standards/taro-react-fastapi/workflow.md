# Codex 开发工作流

## 目标

定义如何依靠 Codex 在本地完成微信小程序项目的主要编码开发工作。

## 推荐工作方式

推荐把开发流程拆成两部分：

- VS Code + Codex：负责绝大多数代码编写、重构、静态检查、测试和命令执行
- 微信开发者工具：负责小程序预览、模拟器观察、真机联调和平台侧确认

## 推荐路径

对于微信小程序项目，建议采用以下流程：

1. 使用脚手架初始化项目
2. 使用 VS Code 打开仓库
3. 让 Codex 在 VS Code 中完成编码
4. 在终端运行前端与后端的本地开发命令
5. 在微信开发者工具中导入项目并预览效果
6. 对需要平台确认的能力进行模拟器或真机测试

## 与 iOS 工作流的对应关系

你之前的 iOS 路径是：

- Xcode 创建项目
- VS Code 中由 Codex 编码
- 编译确认
- 回到 Xcode 模拟器看效果

小程序项目可以对应为：

- 使用 Taro 脚手架初始化项目
- VS Code 中由 Codex 编码
- 运行本地构建、lint、测试命令
- 回到微信开发者工具预览和真机测试

## Codex 适合负责什么

Codex 适合负责：

- 页面和组件开发
- 规则模块实现
- 接口定义与后端实现
- 样式整理
- 测试补充
- 构建错误修复

## 仍需要开发者在平台工具中确认什么

开发者仍应在微信开发者工具中确认：

- 页面真实显示效果
- 小程序路由与生命周期表现
- 平台授权行为
- 真机交互细节
- 平台构建与上传相关问题

## 默认结论

对于这个项目，Codex 不是替代微信开发者工具，而是替代大部分手工编码工作。

更准确的协作方式是：

- Codex 负责写和改
- 终端负责跑和验
- 微信开发者工具负责看和测
